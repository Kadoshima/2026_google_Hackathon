# BE-080：Artifact入力基盤（/v1/artifacts, /v1/capabilities, Adapter土台）

## 担当者
- 萩原

## 指示
- 論文以外の成果物入力（PR/DOC/SHEET）を受け入れるAPIを追加する
- Adapterの差し替えで解析パイプラインが維持される構成にする
- frontendが能力表示できるよう capabilities API を追加する

## 作業ログ
- `POST /v1/artifacts` を実装
  - `artifact_type`, `content`, `content_format`, `source_ref` を受理
  - Firestoreに Session/Submission を記録、Storageに原文保存
- `GET /v1/capabilities` を実装
  - artifact別の `status/supported_inputs/key_checks` を返却
- Artifact Adapter基盤を追加
  - `paper.adapter.ts`
  - `pr.adapter.ts`
  - `text.adapter.ts`
- Orchestratorで `artifactType` による extractor dispatch を実装
