# BE-082：Claim Miner Critic->Refiner 反復ループ導入

## 担当者
- 萩原

## 指示
- Claim抽出を1回生成で終わらせず、品質改善の反復を入れる
- Criticで弱い/重複Claimを判定し、Refinerで置換する
- 反復回数は環境変数で制御する

## 作業ログ
- `ANALYSIS_CLAIM_REFINER_MAX_ITER` を追加
- `buildClaimsWithLlmFallback` を拡張
  - 初期Claim生成（LLM + fallback）
  - Critic（短すぎる/根拠重なりが薄い/重複）
  - Refiner（候補プールから置換）
- デバッグログを追加
  - `llm_claim_critic_iteration`
  - `llm_claim_refiner_iteration`
- Claim Miner の agent summary に反復統計を反映
