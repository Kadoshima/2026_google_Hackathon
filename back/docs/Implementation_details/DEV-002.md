# DEV-002：共通の型共有方針（api schema or shared package）決定＆最小実装

## 担当者
- 萩原

## 指示
- 方針は **shared package で型だけ共有** とする
- ルートに `shared/` パッケージを作り、API request/response 型だけを定義する
- 共有対象は `back` と `front` の両方で import できることを確認する
- 実装は最小（型ファイルのみ）でOK、OpenAPI などは導入しない

## 作業メモ（手順のヒント）
- ルート `package.json` に `workspaces: ["front", "back", "shared"]` を追加するのが簡単
- `shared/package.json` と `shared/tsconfig.json` を作成
- `shared/src/contracts.ts` に API 型を集約し、`shared/src/index.ts` で export
- `back/package.json` と `front/package.json` に `"shared": "workspace:*"` 依存を追加

## 作業ログ
